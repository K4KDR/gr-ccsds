59 /*!
60  * \brief M-PSK Ambiguity resolution.
61  *
62  * \ingroup synchronization
63  *
64  * The input to this block is a stream of unpacked bytes, each containing \c ld(M)
65  * used information bits. The output of this block is a stream of packed bytes.
66  *
67  * The block will start in searchmode, looking for every possible bit and byte
68  * offset in all possible \c M ambiguities. Once an ASM is found it will enter
69  * locked state where it will only check for the ASM at the expected position
70  * and ambiguity. If the ASM is found a counter is increased up to
71  * \c d_THRESHOLD. If an exprected ASM is not found the counter is decreased.
72  * If the counter reaches zero the block goes into search mode again.
73 
74  * The stream with the last known ambiguity and bit offset is send to the output.
75  * In locked state this will result in the correct byte stream beeing output,
76  * while in searchmode the output might be nonsense (since there is no ASM in it)
77  * or might still be correct if the block has lost it's lock to early.
78  */
