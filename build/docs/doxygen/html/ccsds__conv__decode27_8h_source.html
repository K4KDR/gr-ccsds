55 /*! \brief Create a Convolutional decoder block and return it's shared pointer.
56  *
57  * \param gen_poly_c1 Compact generator polynom for c1 stream. Default is 0x79.
58  * \param gen_poly_c2 Compact generator polynom for c2 stream. Default is 0xDB.
59  * \param puncturing_type Symbol for type of puncturing. Default is NONE.
60  * \param block_len Number of output data bits per block as BLOB. (BLOB len =
61  * \c block_len / 8).
62  * \param ASM Unencoded attached sync marker that is used to determine the
63  * start and termination state of the decoder.
64  * \return Shared pointer of the Convolutional encoder block.
65  *
66  * The compact generator polynom contains two informations. Encoded in the MSB
67  * is the information whether the stream should be inverted (MSB set) or not
68  * (MSB unset). Encoded in the 7 less significant bits is the information about
69  * the connections between the shift registers and the output. The MSB-1
70  * represents the influence of the current information bit i(t) while the LSB
71  * represents the influence of the information bit i(t-6). A set bit means
72  * there is a connection, while an unset bit means there is no.
73  *
74  * The polynomials for the convolutional code defined in ECSS can be generated
75  * with 0x79 for c1 and 0x5B for c2. Together with the MSB for the inversion of
76  * c2 the compact polynomials are 0x79 and 0xDB.
77  */

80 /*! \brief Perform unpuncturing and convloutional decoding on the input stream.
81  * \ingroup channelcoding
82  *
83  * This block takes an asynchronous message of punctured convolutional encoded
84  * softbits. The input softbits are frame synchronized, which means that the
85  * leading and following ASM sequences have been stripped off and only contains
86  * the encoded symbols from the actual frame data.
87  *
88  * It performs unpuncturing and viterbi decoding of the message and outputs the
89  * decoded data stream as a BLOB message of packed bytes.
90  *
91  * Since before and after each message a known ASM is send, the knowledge of
92  * this ASM is used to determine the start and and state of the viterbi decoder
93  * as well as the tail sequence of the block, which is the start of the new ASM
94  * sequence that still depends on the data bits of the current block due to the
95  * code's constraint length.
96  *
97  * \todo Synchronize unpuncturing with the puncturer at the sender side. Right
98  * now unpuncturing only works for rate 1/2 and 2/3 as they repeat every
99  * byte.
100  */
