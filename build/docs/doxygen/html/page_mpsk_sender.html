M-PSK modulation chain

Introduction

This pages describes a general purpose M-PSK sender which can be used to transmitt arbitrary M-PSK signals. This block is available as GRC block in apps/blocks/mpsk_mod.grc

This block takes an input stream of packed data bytes and outputs complex M-PSK samples at sampling rate including an optional alternating BPSK preamble to allow the receiver to sync with this sender before sending the real M-PSK data symbols.

Setup

The Figure below shows the GRC block of this M-PSK modulation chain. It's parameters and individual blocks are described in the next Sections.

Setup of the M-PSK modulation chain of this package

Parameters

samp_rate

The sampling rate of the outgoing samples.

M

The Modulation order of the signal. Must be a power of two.

osf

Oversampling factor, or samples per symbol. The sender block performs an upsampling by this factor. Oversampling must be at least two for most receivers to enable syncrhonization.

alpha

Rollof factor from the TX filter, if the root raised cosine filter is used.

preamble_len

Length of the optional BPSK preamble in symbols. If length is zero, no preamble is send.

sym_rate

Symbol rate of the outgoing signal given by samp_rate / osf.

ldM

Number of bits per symbol given by ld(M) with ld beeing the logarithmus dualis (log of base two).

Blocks

Byte unpacker

Takes the packed bytes from the input stream and output bytes with only ldM information bits in each output byte. The ldM bits containing the MSB are taken first, the ones containing the LSB are taken last. The standardt GNURadio Packed to Unpacked block is used.

M-PSK Modulator

The incomming unpacked bytes containing ldM information bits each are transformed into complex baseband symbols. Implemented in mpsk_mod_bc.h.

Preamble

Add preamble_len alternating BPSK symbols, before copying the input stream to the output stream. Implemented in mpsk_preamble_cc.h.

TX filter

The TX filter limits the bandwidth of the send signal and performs the oversampling by a factor of osf. The concatenation of this filter and the RX filter should fulfill the first Nyquist criterion. The simplest TX filter is a rectangular filter, leaving all the filtering to the receiver. The most common solution is to split the filters into equal halves at sending and receiving side and thus using a root raised cosine (RRC) filter at both sides.

These two options are predefined in the block and by default the RRC filter is used. The rectangular filter can be used by enabling the Low Pass Filter and disabling the Root Raised Cosine Filter.

Todo:
In the future this block should be able to switch between the two filters and leaving the GRC definition untouched.

Root Raised Cosine Filter

A standard GRC RRC filter implementation is used.

Rectangle Lowpass Interpolation Filter

A standard GRC filter implementation is used.
