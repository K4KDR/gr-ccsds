35 /*! \brief De/randomise a TM Frame using an LFSR.
36  * \ingroup ccsds
37  *
38  * \param output_queue The message queue where to output individual messages (frames) after processing
39  * \param frame_bits The size of the frame in bits
40  * \param mask Polynomial mask for LFSR
41  * \param seed Initial shift register contents
42  * \param len Shift register length
43  *
44  * The input is a stream of unpacked bits that represent the encoded frames only,
45  * i.e. the ASMs have been removed from the stream, and are synchronised on the first bit of the frame.
46  *
47  * There is no stream output, instead the bits of the decoded frame are packed
48  * and inserted into a dedicated output queue.
49  *
50  * The scrambler works by XORing the LSBs of the incoming bit stream by the output of
51  * the LFSR. After 'frame_bits' bits have been processed, the shift
52  * register is reset to the seed value. This allows processing fixed length
53  * frames according to CCSDS recommendations.
54  *
55  * The gnuradio-core gri_lfsr has a bug in that can be worked-around by defining the length of the
56  * shift register (in the 'len' parameter) to be one less than its actual length
57  *
58  * \ingroup telemetry
59  */
