67 /*!
68  * \brief M-PSK soft bit ambiguity resolution and frame synchronization.
69  *
70  * \ingroup synchronization
71  *
72  * The input to this block is a stream of \c d_ldM grouped soft bits. The output
73  * of this block is a asynchronous message containing a float vector with the
74  * soft bits of a frame. The bytes passed to this block as ASM are cut off, but
75  * any skipped bits will be in the output message.
76  *
77  * So if you pass the two ASM bytes 0x00 0xAA to this block followed by two
78  * bytes frame data (FD1 and FD2), followed by another two bytes ASM and set
79  * \c asm_skip_first_bits to zero the output will be a message containing FD1
80  * and FD2. If \c asm_skip_first_bits is set to two, the output will be FD1, FD2
81  * followed by the first two zero bits from the ASM.
82  *
83  * Using \c asm_skip_first_bits (setting it to nonzero values)
84  *
85  * The block will start in searchmode, looking for every possible bit offset
86  * in all possible \c M ambiguities. Once an ASM is found it will enter
87  * locked state where it will only check for the ASM at the expected position
88  * and ambiguity. If the ASM is found a counter is increased up to
89  * \c d_THRESHOLD. If an exprected ASM is not found the counter is decreased.
90  * If the counter reaches zero the block goes into search mode again.
91  */
