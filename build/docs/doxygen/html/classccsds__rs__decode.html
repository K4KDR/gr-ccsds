Detailed Description

Takes the transmitted codeblock and generates the frame out of it, or drops it, if recovery is not possible.

The block has an input message port named "in" at which codeblocks of length 255*I bytes are received. On reception the data codeblock is interleaved and decoded. The generated frame consisting of 223*I bytes is then queued in the output message port named "out" or dropped if it can't be decoded.
