Detailed Description

M-PSK soft bit ambiguity resolution and frame synchronization.

The input to this block is a stream of d_ldM grouped soft bits. The output of this block is a asynchronous message containing a float vector with the soft bits of a frame. The bytes passed to this block as ASM are cut off, but any skipped bits will be in the output message.

So if you pass the two ASM bytes 0x00 0xAA to this block followed by two bytes frame data (FD1 and FD2), followed by another two bytes ASM and set asm_skip_first_bits to zero the output will be a message containing FD1 and FD2. If asm_skip_first_bits is set to two, the output will be FD1, FD2 followed by the first two zero bits from the ASM.

Using asm_skip_first_bits (setting it to nonzero values)

The block will start in searchmode, looking for every possible bit offset in all possible M ambiguities. Once an ASM is found it will enter locked state where it will only check for the ASM at the expected position and ambiguity. If the ASM is found a counter is increased up to d_THRESHOLD. If an exprected ASM is not found the counter is decreased. If the counter reaches zero the block goes into search mode again.

Search for ASM in input sequence and return maximum found correlation.
